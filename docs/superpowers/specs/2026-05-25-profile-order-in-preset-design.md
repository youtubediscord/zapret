# Порядок profile-ов внутри preset

## Задача

На странице настройки preset-а есть основной список profile-ов с удобной группировкой: YouTube, Discord, мессенджеры и другие разделы. Этот экран остаётся основным и не должен объяснять порядок запуска, потому что его задача — удобно показать все доступные profile-ы и настройки.

Нужна отдельная опция `Порядок в preset`, которая показывает только реальные profile-ы текущего preset-а в том порядке, как они записаны в `.txt` файле.

## Поведение в обычной настройке preset-а

Обычная страница остаётся режимом по умолчанию.

- Показываются группы profile-ов, как сейчас.
- Показываются profile-ы из preset-а и profile-ы, которые ещё можно добавить.
- Перетаскивание внутри групп меняет только удобную раскладку в интерфейсе.
- Порядок блоков внутри `.txt` файла preset-а не меняется.
- Дополнительный текст про приоритет profile-ов на этой странице не показывается.

## Поведение в `Порядок в preset`

`Порядок в preset` открывается отдельной кнопкой на странице настройки preset-а. Это не второй постоянный режим рядом с группировкой, а отдельная вложенная опция для одной задачи.

Внутри этой опции:

- список плоский, без групп и папок;
- показываются только profile-ы, которые реально есть в текущем preset;
- profile-ы со статусом `Не добавлен` не показываются;
- лишние действия у строк убираются, чтобы экран был только про перестановку порядка;
- перетаскивание profile-а выше или ниже другого profile-а физически меняет порядок блоков в `.txt` файле preset-а;
- после сохранения список перечитывается из файла, чтобы пользователь видел реальный сохранённый порядок.

На этом экране показывается пояснение:

> Profile выше в списке имеет больший приоритет. Если два profile-а подходят к одному домену или IP, будет применён тот, который находится выше.

## Техническая цепочка

Основная страница продолжает использовать текущий список profile-ов и текущую группировку через состояние папок в `settings.json`.

Для `Порядок в preset` нужен отдельный путь данных:

1. Прочитать выбранный preset.
2. Распарсить его в список profile-ов.
3. Построить плоский список только из `preset.profiles`.
4. При drag/drop вызвать сервисный метод перестановки profile-а.
5. Сервис меняет порядок profile-ов в модели preset-а.
6. Сериализует preset обратно в `.txt`.
7. Инвалидирует кэш profile-списка.
8. UI перечитывает список из файла.

В GUI страница только показывает список и вызывает готовое действие сервиса. Чтение, запись и перестановка блоков остаются в profile/preset-сервисе.

## Ошибки и ограничения

Если preset не удалось прочитать или сохранить, показывается обычная ошибка через InfoBar, а текущий список не должен делать вид, что порядок успешно изменился.

Если пользователь перетащил profile на то же место, файл не переписывается.

Если profile исчез из preset-а во время работы экрана, список перечитывается из файла.

## Проверка

Нужны тесты на три уровня:

- сервис переставляет profile-ы и сохраняет корректный порядок `--new` блоков;
- плоский список `Порядок в preset` не содержит profile-ов `Не добавлен`;
- drag/drop в обычной группировке продолжает менять только интерфейсную раскладку, а drag/drop в `Порядок в preset` меняет файл preset-а.
